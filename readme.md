# Core

## Project structure 

Пока что используется вариация чистой архитектуры. Возможно, потом перейдем на модульную, но для этого надо хорошо понимать язык (чего у нас пока нет),
а так же хорошо понимать домен (пока чего так же нет).

```
backend-core/
├─ gradle/ - gradle stuff. Нужен для настройки gradle, лучше не трогать
├─ src/
│  ├─ main/kotlin/ru/poomsae/core/
│  │  ├─ adapter/ - здесь будут лежать все, с помощью чего мы общращаемся к чему-то: клиенты, базы, брокеры и т.д
│  │  │  ├─ repository/ - здесь расположены интерфейсы репозиториев и их реализации
│  │  ├─ config/ - конфиг приложения
│  │  ├─ domain/ - здесь будут лежать все сущности, которыми будем манипулировать
│  │  │  ├─ model - здесь находятся сущности бизнес логики
│  │  ├─ driver - здесь будут лежать все, с помощью чего обращаются к нашему сервису: http-роутер, grpc-роутер, сабскрайберы и т.д
│  │  │  ├─http/dto
│  │  ├─ controller - здесь лежат REST-контроллеры
│  │  ├─ mapper/ - MapStruct мапперы (Domain <-> DTO)
│  │  ├─ service/ - здесь лежат интерфейсы бизнес-логики и их реализации
│  │  ├─ controller/ - здесь лежит реализация интерфейсов бизнес-логики
│  │  ├─ CoreApplication.kt - стартовая точка приложения
│  ├─ test/kotlin/ru/poomsae/core/ - вроде как тесты, автоматические нагенерило, пока не понял
├─ build.gradle.kts - так же файл конфигурации gradle, тут конфигурируются плагины и всякое такое
```
